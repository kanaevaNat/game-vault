#  Game Vault

**Game Vault** — это SPA-библиотека видеоигр и игровых новостей, созданная на Vue 3 + Vite. Проект разработан как pet-проект для демонстрации навыков проектирования архитектуры, работы со сложным UI и управления состоянием.

## **Демонстрация** 

### Проект развёрнут и доступен по ссылке: [Ссылка на проект](https://www.red-dragon-core.ru/) 

##  Ключевые особенности проекта

- **Кастомная система фильтрации:** На странице каталога игр реализована сложная система фильтров (по году, издателю, студии, категориям) и поиска, написанная **полностью с нуля** без использования UI-библиотек.
- **Демонстрационная Админ-панель:** Эндпоинт `/admin` открыт для просмотра без авторизации (Read-Only режим). Это сделано специально для демонстрации функционала CRUD-таблиц.
- **Высокая абстракция Admin-панели:** Вместо создания десятков однотипных страниц, я реализовала универсальные компоненты, которые адаптируются под любую сущность:
    * **`EntityForm`** — единая форма, которая динамически перестраивает свои поля (input, select, textarea) в зависимости от передаваемой конфигурации сущности.
    * **`EntityTable` & `EntityList`** — универсальные компоненты для отображения данных и управления ими.
    * Это позволило масштабировать админку (добавлять новые сущности типа "Студии" или "Страны") просто добавляя схему данных.
- **Гибридный UI-подход:** Клиентская часть приложения написана на кастомных компонентах с использованием SCSS, в то время как Админ-панель собрана на базе Vuetify 3.

##  Страницы и маршруты (Endpoints)

Приложение построено на базе Vue Router и включает в себя следующие основные страницы:

* **`/` (Главная страница)** — Приветственный экран (`HomeInformation`), превью последних новостей и игр и подписка на рассылку.
* **`/games` (Каталог игр)** — Вывод карточек игр в виде сетки с пагинацией и кастомной панелью фильтров (`FiltersGrid`, `FilterSelect`, `FilterRange`).
* **`/games/:id` (Детали игры)** — Страница с подробной информацией о конкретной игре.
* **`/news` (Лента новостей)** — Полноценная новостная лента карточного типа.
* **`/news/:id` (Статья)** — Чтение конкретной новости. Текст статей хранится в Markdown и парсится.
* **`/admin` (Панель управления)** — Страница для управления контентом (CRUD для игр, новостей, студий, издателей, категорий и стран).
* **`/:pathMatch(.*)*` (404 Not Found)** — Кастомная страница обработки несуществующих маршрутов.
* Также на всех страницах доступна модальная форма авторизации (`LoginModal`).

##  Архитектура и структура файлов

Проект организован с использованием принципов модульной архитектуры (близко к Feature-Sliced Design), что позволяет масштабировать приложение и переиспользовать логику:

```text
src/
├── entities/           # бизнес-сущности
│   ├── admin/          # логика и компоненты авторизации/админки
│   ├── category/       # списки и сторы категорий
│   ├── country/        # списки и сторы стран
│   ├── games/          # всё что связано с играми (карточки, фильтры, страницы, store)
│   ├── news/           # новостной модуль (лента, превью)
│   ├── publisher/      # списки и сторы издателей
│   └── studio/         # списки и сторы студий разработки
│
├── shared/             # переиспользуемый код и глобальные компоненты
│   ├── api/            # настройка axios
│   ├── components/     # глобальные UI компоненты (Header, Footer, MainLayout, Search)
│   ├── composables/    # переиспользуемые хуки (useStats, useTableState)
│   ├── icons/          # SVG иконки
│   ├── styles/         # глобальные SCSS файлы (variables, mixins, reset, markdown)
│   └── utils/          # утилиты (форматирование дат, сборка FormData)
│
├── plugins/            # плагины (Vuetify, Vuelidate)
├── router/             # Vue Router
├── App.vue             # корневой компонент
└── main.js             # точка входа
```

## Запуск проекта локально
Для запуска проекта убедитесь, что у вас установлен Node.js (рекомендуется версия 18+) или Docker.

**Вариант 1**: 

Через Node.js 

1. Склонируйте репозиторий.

2. Скопируйте `.env.example` в `.env` и подставьте нужные значения API.

3. Установите зависимости:

```Bash
npm install
```
Запустите сервер для разработки:

```Bash
npm run dev
```

**Вариант 2**: 

Через Docker (Для Production/Demo)
Приложение полностью готово к запуску в контейнере (включает настроенный `nginx.conf`).

```Bash
docker-compose up -d --build
```
